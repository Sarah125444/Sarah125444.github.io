BOM 2019/08/09

BOM的基本概念

Javascript分三个部分：

  1. ECMAScript标准—基本语法
  2. DOM—–Document Object Model 文档对象模型，操作页面元素
  3. BOM—–Browser Object Model 浏览器对象模型,操作浏览器

浏览器中有个顶级对象：window
页面中有个顶级对象：document
页面中所有的内容都是属于浏览器的，页面中的内容也都是window的
变量是window的

系统的对话框

系统的对话框是不用的，因为在每个浏览器解析都不一样，不受控制，如果要这个功能，就自己写。

页面加载事件

window.onload — js文件最好放在最后面，这样子页面加载的时候比较快

页面加载的时候，按钮还没出现，页面没有加载完毕
页面加载完毕再获取按钮
只要页面加载完毕，这个事件就会触发—-页面中所有的内容，标签，属性，文本，包括外部引入js文件

BOM中有两个定时器—-计时器

小结

为元素绑定事件的三种方式：

  1. 对象.on事件类型 = 事件处理函数；
  2. 对象.addEventListenter( “没有on的时间类型”,事件处理函数, false); //IE8不支持
  3. 对象.attachEvent( “有on的事件类型” , 事件处理函数);

为元素解绑事件的三种方式：

  1. 对象.on事件类型 = null;
  2. 对象.removeEventListener(“没有on的事件类型”,事件处理函数);
  3. 对象.detachEvent(“有on的事件类型”,事件处理函数);

事件冒泡：元素A中有元素B,都有相同的事件，里面的元素的事件触发了，外面元素的事件也会触发，可以是多个元素嵌套。

阻止事件冒泡：
两种：

  1. e.stopPropagation();e—事件参数对象，谷歌和火狐支持，IE8不支持。
  2. window.event.cancelBubble=true;
    window.event就是事件参数对象—e是一样的

BOM：Browser Object Model 浏览器对象模型—操作浏览器
历史记录的后退和前进 history: back()后退 forward()前进
地址栏上的地址的操作 location: href属性跳转页面，assgin()跳转页面，reload刷新,replace()方法替换地址栏上的地址，没有历史记录的
获取系统和浏览器的信息navigator / userAgent属性—-获取系统，浏览器的信息

地址栏上#及后面的内容
console.log(window.location.hash);
主机名及端口号

console.log(window.location.hostname);
文件的路径—相对路径

端口号

定时器：
该定时器会返回自己的id值

  1. var timeId=window.setInterval(函数,时间);
    执行过程：页面加载完毕后，过了一段时间执行函数，这样是反复的，除非清理定时器
    window.clearInterval(定时器的id值);//清理定时器
  2. var timeId=window.setTimeout(函数,时间);
    只执行一次就结束
